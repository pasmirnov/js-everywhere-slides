Ты что несешь вообще?

  • Node.js - среда выполнения, основанная на JavaScript движке Chrome v8.
  • Работает в одном потоке.
  • Параллелит с помощью событийно-ориентированной модели (event loop).
  • Использует неблокируещий ввод-вывод.
  • Есть свой менеджер пакетов - Node Package Manager, npm.

Node.js: В чем хорош?

  • Хранилища файлов.
  • Стриминговые сервисы.
  • Чаты, торговля на бирже, системы мониторинга. "Real-time" штуки.
  • Прокси, балансировщики нагрузки.
  • Изоморфные приложения.
  • Да просто серверы.

Как дела у ноды?

  • Уже 7 лет в деле (с 2009, npm since 2010).
  • v7.9.x (v6.10.2 LTS)
  • 436k модулей в npm (~1100 каждый день)
  • Народ использует: Netflix, Uber, NASA, PayPal, LinkedIn и еще куча

Рост числа модулей

Node.js: Чем плох?

  • Не подходит для выполнение "тяжелых" вычислений.
  • Очень много библиотек.
  • Масштабируемость требует дополнительных шаманств.
  • Язык с динамической типизацией сложнее поддерживать.

Десктопные приложения


Electron: Хорошо

  • CSS
  • Темы
  • Кросс-платформенность
  • Переиспользование JavaScript кода
  • Отзывчивый UI

Electron: Плохо

  • CSS
  • Нет поддержки "look-and-feel" операционной системы
  • Сравнительно недавно вышел

Ненативные мобильные приложения

Cordova: Хорошо

  • Одно и то же приложение под Android и iOS.
  • Переиспользование JS из Web-App.
  • Переиспользвание людей.

Cordova: Плохо

  • Медленно (особенно на Android).
  • Нет нативного "look-and-feel".
  • Некоторые нативные фичи недоступны.
  • С плагинами все плохо.

Нативные мобильные приложения

Native Frameworks

  • React Native
  • Native Script
  • Fuse
  • Tabris

React Native: Хорошо

  • Переиспользование JS кода.
  • Все шустро на iOS.
  • Можно использовать нативные фичи, библиотеки и компоненты интерфейса.
  • Единая комманда разработчиков.

React Native: Плохо

  • Все еще медленно на Android.
  • Нужно реализовывать два отдельных представления под Android и iOS.
  • До некоторых нативных фич тяжело добраться без костылей.
